# 实板冒烟验证

这份流程用于把网页模拟结果升级为真实开发板证据。任何一步出现发热、异味、异常电流、接反或不确定接线，立即断电并停止。

## 开始前

- 开发板：STM32F103C8T6；下载器：ST-LINK V2；串口：3.3 V TTL。
- 所有模块与开发板共地，先断电接线，再由第二次目视检查确认 3.3 V、GND 和信号线。
- 禁止把 5 V 直接接入 GPIO；舵机和直流电机使用独立电源及驱动器，不由 GPIO 直接供电。
- W25Q64 往返测试会临时修改一个扇区，但固件会先备份、测试后恢复；掉电前确认恢复通过。

## 十步安全流程

1. 断电检查：核对芯片型号、跳帽、供电电压、GND 和所有模块方向，拍下接线全景。
2. 仅连接 ST-LINK，烧录 `device-test-v1.hex`；确认下载成功、开发板无异常升温。
3. 接入 3.3 V USB 转串口，网页完成握手、芯片 ID、断线重连；保存串口和端口说明。
4. 断电增加 GPIO 回环线，运行 GPIO 与 EXTI 检查；记录实际高低电平和事件计数。
5. 断电增加 PWM 捕获回环线，运行 PWM 检查；记录频率、占空比和允许误差。
6. 断电接入 MPU6050 与 W25Q64，先只读设备 ID；型号不匹配时停止，不运行写入测试。
7. 仅在 W25Q64 ID 精确匹配后运行往返写入；确认备份、写入、校验、恢复和恢复后全扇区校验均成功。
8. 运行 RTC/BKP、看门狗和内部 Flash 检查；看门狗应发生预期复位，Flash 保留页必须完成恢复校验。
9. 分别人工观察 LED、蜂鸣器、舵机/电机和低功耗唤醒；每次改变接线前断电，记录可观察现象。
10. 串联电流表测量低功耗电流，完成后导出网页报告；把每项实际数值、时间、接线说明和照片路径写入证据文件。

## 通过规则

网页模拟器只能用于预演，不能计作实板通过。每个通过项必须包含真实设备或人工观察来源、实际结果、时间和接线说明；缺任何一项都保持“待验证”。

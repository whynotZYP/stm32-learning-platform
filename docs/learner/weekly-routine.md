# 每周固定学习循环

每周按同一个九步循环完成。这样做的目的不是赶进度，而是留下能复现的理解和实验证据。

## 九个步骤

1. **预检**：阅读本周安全条目；断电检查电源、地线和引脚；导出一次进度备份。
2. **理解**：读 `curriculum/weeks/wNN.md`，用自己的话解释本周概念，不会的术语记到笔记。
3. **配置**：在 STM32CubeMX 打开本周 `.ioc`，逐项对照时钟、引脚和外设参数。
4. **构建**：在对应固件目录运行 `cmake --preset Debug --fresh` 与 `cmake --build --preset Debug --parallel`。
5. **观察**：烧录后记录串口数值、逻辑电平或实际现象；自动数据和亲眼观察分开写。
6. **故意破坏**：只执行课程指定的安全故障，例如拔掉一根信号线；不要短接电源或绕过限流器件。
7. **解释**：根据现象、寄存器和代码路径说清楚故障发生在哪里，而不只写“修好了”。
8. **考核**：完成本周测验或阶段门槛；未通过时按“优先补强”完成补修，再重新评估。
9. **笔记**：保存接线、参数、实际值、失败原因和下一步，并按 [VS Code 保存学习笔记](github-notes.md) 提交。

## 每周结束标准

- 构建命令退出码为 0，且固件大小没有超出开发板 Flash/RAM。
- 自动项目有原始数值；声音、动作、电流等项目有人工观察。
- 失败实验有“预期—实际—定位—修复—复测”记录。
- 网页进度已经保存并导出备份。实机没做就保留“待验证”，不要填成通过。

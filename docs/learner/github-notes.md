# 用 VS Code 保存学习笔记

Git 是文件版本记录工具；一次 `提交`相当于给当前改动做一个有名字、可追溯的存档。

## 保存一次学习记录

1. 在网页“笔记与备份”里保存本周笔记，并把需要长期保存的内容整理到仓库的 Markdown 笔记。
2. 点 VS Code 左侧像分叉线的“源代码管理”图标。
3. 查看“更改”列表，逐个点开。绿色是新增、红色是删除；确认只包含这次学习内容和预期生成文件。
4. 在消息框写清楚内容，例如 `notes: record week 5 gpio input results`。
5. 点“提交”。如果 VS Code 提示先暂存，选择暂存本次确认过的文件。
6. 点“同步更改”，把本地提交传到 GitHub。没有远程仓库时，本地提交仍然有效。

## 何时停止

看到“冲突”时不要随便选“接受全部”。先保留双方文件和网页 JSON 备份，再请人逐段比较。不要把含密码、令牌、私人路径或大型构建缓存的文件提交到 GitHub。

网页进度的 JSON 备份与 Git 提交作用不同：前者保存学习状态，后者保存仓库文件。两者都要做，具体见 [备份与恢复](backup-restore.md)。

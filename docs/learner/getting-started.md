# 第一次使用 STM32 学习平台

这份指南假定你没有编程经验。`编译`就是把人写的代码变成芯片能运行的文件；`烧录`就是把这个文件写进开发板。

## 先准备什么

- STM32F103C8T6 开发板、ST-LINK 下载器、CH340 3.3 V 串口模块、面包板和跳线。
- 后续外设实验再准备 MPU6050、W25Q64、LED 与限流电阻、蜂鸣器、舵机或电机驱动模块。
- Windows 电脑、VS Code、STM32CubeMX、STM32CubeCLT、Node.js 22。

安装与检查命令见 [Windows 工具链说明](../setup/windows-toolchain.md)。接线前先断开 USB 和外部电源；STM32 普通引脚只接 0–3.3 V，不能直接接 5 V。

## 第一次打开

1. 用 VS Code 打开仓库文件夹。
2. 打开“终端”，依次运行 `node --version`、`npm --version`、`cmake --version`、`arm-none-eabi-gcc --version`。每条都应显示版本号。
3. 运行 `npm ci`，再运行 `npm run dev`。
4. 按终端给出的本地网址打开网页，点“继续学习”与“进入本周课程”。
5. 完成“入门诊断”。不确定的开放题如实写，不要为了分数填满分。
6. 打开“笔记与备份”，先点“导出全部进度”，把 JSON 备份放到自己认识的文件夹。

## 什么时候算完成

网页的“自动通过”只证明对应数据；“待人工确认”说明还需要亲眼观察或实板测量。模拟器永远不能替代实机证据。遇到待确认项可以停在这里，先读接线说明或请有经验的人复核，不要带电反复试线。

# 第 18 周：硬件 I2C 读取 MPU6050 六轴数据

## 学完后能解释

MPU6050 上电后先向 `PWR_MGMT_1(0x6B)` 写 0 唤醒，再读取 `WHO_AM_I(0x75)`，默认地址 AD0=0 时应得到 `0x68`。六轴数据从 `ACCEL_XOUT_H(0x3B)` 起连续读取 14 字节：三轴加速度、温度、三轴角速度。每个测量值是高字节在前的二补码。安全组合顺序是先用 unsigned `uint16_t` 执行 `(high << 8) | low`，最后一次性 cast 为 `int16_t`；过早把高字节当有符号数移位会引入实现相关行为或符号扩展错误。

默认 ±2 g 时加速度灵敏度为 16384 LSB/g，默认 ±250 °/s 时陀螺仪为 131 LSB/(°/s)。本工程输出 mg 与 mdps，并在换算前减去 bias。滤波器不是“创造精度”，只把目标值限制在 ±2000 mg、±250000 mdps 后做四分之一步的一阶平滑，保证异常输入不会让输出无限增大。偏置应由静止校准得到；示例零偏只是代码路径，不代表传感器已经校准。

## 学完后能做到

能依据制造商 register map 找到唤醒、身份与连续数据寄存器；能用 I2C1 有界 HAL 调用读取 14 字节；能正确处理 `0x7FFF`、`0x8000`、`0xFFFF` 等 signed vectors；能把 raw 值减 bias 后按量程换算，并解释简单 bounded filter 的响应速度与局限。能区分“主机 C 测试证明字节组合正确”和“实机静止数据证明接线与器件工作”。

## 概念模型

把寄存器连续区看成一排按高、低字节装订的测量卡。读出两张卡时先拼成 16 位无符号图案，再把同一组位解释成有符号二补码。scale 是尺子的刻度密度，bias 是尺子零点偏移，filter 是让显示不要追随每个瞬时抖动的缓冲器。三者顺序不能颠倒：字节先正确组合，随后减偏置并缩放，最后滤波。

## CubeMX 为什么这样配

I2C1 使用 PB6/SCL、PB7/SDA、100 kHz、7 位地址模式，GPIO 为复用开漏，外部上拉接 3.3 V。`HAL_I2C_Mem_Write` 和 `HAL_I2C_Mem_Read` 都使用固定 10 ms timeout；采样由 `HAL_GetTick` 每 20 ms 调度，不用 `HAL_Delay`，也不在 IRQ 中做循环读取。连续读 14 字节比逐寄存器读取减少 START/STOP 次数，并保证同一采样帧的六轴数据相邻。

## 最小实验

1. 断电后接 3.3 V、GND、PB6/SCL、PB7/SDA，AD0 接地，确认所有模块共地且上拉不超过 3.3 V。
2. 下载 `w18-mpu6050`，先观察 `who_am_i`；只有 HAL 返回 OK 且值为 `0x68` 才进入采样。
3. 板子静止平放，观察一轴接近 ±1000 mg、其余轴接近 0 mg，角速度接近 0 mdps；方向取决于器件坐标轴。
4. 缓慢转动并记录 raw、scaled、filtered 三组数据，确认 filtered 逐步追随而不是越界。
5. 若没有真实板卡或传感器，保留主机 signed vectors 结果并把物理观察标记为“待实机验证”。

## 调试与寄存器观察

先检查 I2C 返回状态与 `WHO_AM_I`，再查 `PWR_MGMT_1` 是否清除了 SLEEP。若所有轴固定为 0，核对起始寄存器 `0x3B` 和连续长度 14；若正负颠倒，单独验证 `0x8000 -> -32768` 与 `0xFFFF -> -1`；若数值相差 2 或 4 倍，检查 `ACCEL_CONFIG.AFS_SEL`、`GYRO_CONFIG.FS_SEL` 与 scale factor 是否一致。观察 I2C1 的 AF/BERR/BUSY 只用于定位，不能替代数据手册的读取顺序。

## 故障注入

在主机测试中交换一组高低字节，确认已知向量立即失败；把 ±2 g 的 16384 错改为其他量程系数，检查 1 g 向量不再得到 1000 mg。实机上可断开 SDA 制造 NACK，应用必须在 10 ms 上限内返回，不能永久卡住。任何接线更改先断电，不向模块施加 5 V。恢复后重新验证身份、静止方向和有界滤波输出。

## 复述检查

为什么要先组合 `uint16_t` 再 cast `int16_t`？`0x80 0x00` 应是多少？±2 g 和 ±250 °/s 的 scale factor 分别是多少？bias 在缩放前还是后应用？一阶四分之一滤波会怎样影响突变？`WHO_AM_I=0x68` 能证明什么，又不能证明什么？哪些结论必须留作待实机验证？

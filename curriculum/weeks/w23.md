# 第 23 周：可靠数据记录器综合项目

最终固件不是内存演示：它通过 STM32 HAL 实际连接 ADC 光照/温度、I2C MPU6050、RTC UTC、SPI W25Q64 日志和 USART health 显示。改接所有模块前必须断电，模块统一 3.3 V 并共地。当前外设读数与断电恢复全部待实机验证。

## 学完后能解释

- `Sensors` 负责 ADC 与 MPU6050 快照，`Storage` 负责记录格式和 W25Q64，`Clock` 提供 UTC，`Display` 输出 health，`Health` 判断进展，`App` 只编排状态。
- 记录包含 `magic`、`version`、`length`、timestamp 与 `checksum`；坏记录被跳过，不把后续好记录一起丢弃。
- 内部 capstone settings 保留在 `0x0800F800`，外部 W25Q64 保存日志；两者职责不同。

## 学完后能做到

- 顶层只推进 `CAPSTONE_STATE_SAMPLE -> STORE -> DISPLAY -> WAIT`，不使用 RTOS、`HAL_Delay`、`osDelay` 或 `vTaskDelay`。
- `SensorsHal_Sample` 真实调用 `HAL_ADC_Start/Poll` 与 `HAL_I2C_Mem_Read`；`StorageW25Q64_Append` 真实执行 read、write-enable、page-program、busy timeout 与 readback。
- 只有 sample、storage、display 三个 progress bit 都完成后，`Health_TakeFeedPermission` 才给一次 IWDG refresh。

## 概念模型

纯 C 模块定义可测试规则，HAL adapter 只翻译硬件动作。生产 `storage.c` 同时被 host test 与 MCU 固件编译；`storage_w25q64.c` 是真实 SPI 驱动，不是空 stub。日志使用 32-byte slot，启动扫描第一个 erased slot；正常 append 不自动 sector erase，显式 `StorageW25Q64_EraseSectorRequested` 才允许擦除。

## CubeMX 为什么这样配

ADC1 使用 PA0/PA1 长采样时间；I2C1 PB6/PB7 100 kHz 读取 MPU6050；SPI2 PB13/PB14/PB15 配 W25Q64，PB12 为 CS；USART1 PA9/PA10 115200 输出 health，避免与 ADC 引脚冲突；RTC 选择 LSE 和 32767。链接 FLASH 长度 62K，保留 `0x0800F800` 与 `0x0800FC00` 两页，避免程序覆盖 settings/实验保留页。

## 最小实验

1. host behavior 构造两条记录，破坏第一条 checksum，扫描必须返回第二条的 offset 和 UTC。
2. 依次推进 SAMPLE、STORE、DISPLAY；任一步失败都不喂狗。
3. mutation 把 required mask 改成只需 SAMPLE，编译后的测试必须运行失败。
4. 构建 ELF，确认 CMake 同时编译六模块与五个 HAL adapters。
5. 实机采样、W25Q64 追加和重启扫描当前待实机验证。

## 调试与寄存器观察

观察 ADC conversion 状态、I2C 地址 `0x68<<1` 与 accel register `0x3B`、SPI2 busy/status、W25Q64 status bit0、RTC CNTH/CNTL、USART 返回状态和 Health mask。每个 HAL 调用都有有限 timeout；失败时当前状态不前进，因此不会假装健康后喂狗。

## 故障注入

- 将首条 checksum 最后一个字节翻转，扫描应跳过损坏记录。
- mutation 删除 STORAGE/DISPLAY 进展要求，测试必须拒绝。
- 拔掉 MPU6050、填满 W25Q64 slot 或让 UART 超时，系统不应喂狗；这些设备结果待实机验证。
- sector erase 只接受 4KB 对齐的显式请求，禁止在普通 append 中自动擦除旧日志。

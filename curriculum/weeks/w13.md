# 第 13 周：DMA 数据搬运与 ADC 循环扫描

## 学完后能解释

你能解释 DMA（Direct Memory Access，直接存储器访问）为什么不是“更快的 CPU 循环”，而是一条由外设请求触发、独立完成地址读写的数据通路。一次传输必须说清请求来源、peripheral-to-memory 或 memory-to-memory 方向、源/目标地址是否递增、每项宽度、总计数和优先级。DMA 完成后 CPU 仍要判断完成标志和结果，DMA 不负责理解数据含义。

你还能解释 normal 与 circular：normal 计数归零便停，circular 会重新装载地址和计数。ADC scan 固定按 rank 产生四个 halfword；DMA 的 peripheral/memory width 都必须是 halfword。宽度错配不是“数值差一点”，而会改变地址步长与缓冲布局。

## 学完后能做到

先用 DMA1_Channel2 搬运四个 32-bit word，逐项比对目标，建立最小闭环。再让 ADC1 扫描 PA0–PA3，DMA1_Channel1 以 circular 模式写入两个连续四通道帧。半传输完成时发布前四项，全传输完成时发布后四项。

应用不得直接把正在更新的 DMA 数组交给显示或计算。`DmaSnapshot_Publish` 先完整复制到未发布槽，再一次切换槽号；`DmaSnapshot_Read` 只有在序号和槽号前后一致时才返回，从而避开“通道 0 是新帧、通道 3 还是旧帧”的半更新。

## 概念模型

CPU 搬运像人逐箱搬货：load、store、计数、判断都占 CPU 指令。DMA 像按清单工作的搬运机：CPU 配地址/宽度/计数并启动，ADC 每次 EOC 发 request，DMA 完成一次 halfword 搬运并递减计数。HT/TC 中断只是“半批/整批完成”的通知，不应在回调里做显示、格式化或长运算。

四通道、两帧缓冲共有 8 个 `uint16_t`。DMA 写 0–3 后产生 half complete，此时 0–3 已稳定而 4–7 尚未成为新帧；写 4–7 后 complete。稳定快照 API 把“何时可读”变成明确合同。

## CubeMX 为什么这样配

目标是 STM32F103C8Tx、HSE 8 MHz、系统 72 MHz。ADC 时钟为 12 MHz，PA0–PA3 配 analog，scan 为 4 ranks、continuous enable。ADC1 固定映射 DMA1_Channel1，所以该通道为 peripheral-to-memory、peripheral increment disable、memory increment enable、两端 halfword、circular、high priority，并开启 Channel1 IRQ。

memory-to-memory 使用独立 DMA1_Channel2，word 对齐、两端递增、normal mode；它不与 ADC 抢同一 channel。所有轮询都有 10 ms 上限，避免配置错误后永久卡住。

## 最小实验

第一步不接模拟信号，只启动 memory DMA：源数组为四个可辨认的 32-bit 常量，目标先清零，完成后逐项比较。第二步断电，把四个 0–3.3 V 信号接 PA0–PA3 并共地，再启动 ADC calibration 一次和 8-halfword circular DMA。

观察 `App_MemoryDmaPassed()`、`App_GetAdcSnapshot()`、snapshot sequence 和四个 values。逐路改变输入时只对应 rank 改变；sequence 只在完整四项发布时增长。自动构建不能证明电压正确，实机读数仍要用万用表比对。

## 调试与寄存器观察

查看 DMA1 `CCR1` 的 DIR、CIRC、PINC、MINC、PSIZE、MSIZE，`CNDTR1` 应从 8 递减并循环装载；`CPAR1` 指向 ADC1 DR，`CMAR1` 指向缓冲。检查 ISR 的 HTIF1/TCIF1 与 HAL 回调顺序。ADC 侧查看 SQR 的 rank 顺序和 CR2 的 DMA/CONT。

若快照序号停止，先区分 ADC 没请求、DMA 没启动、IRQ 没开和回调没有匹配 ADC1。不要靠随意增加 delay 掩盖配置错误。

## 故障注入

把 memory width 改成 byte，同时保持 ADC halfword 数据，记录缓冲排列为何错误；随后恢复两端 halfword。再把 circular 改 normal，观察一轮后 `CNDTR1` 为 0 且 sequence 不再增长。最后故意读取原始 DMA buffer 与稳定快照比较，说明为什么偶发“看似正常”不能证明没有竞争。

任何改线前必须断电，输入不得低于 0 V 或高于 3.3 V，信号源与开发板必须共地。

## 复述检查

不用看代码回答：谁产生 DMA request？为什么 ADC1 不能随意换 DMA channel？四通道为何计数是 8 而不是 4？PSIZE/MSIZE 错了会改变什么？HT 与 TC 各保证哪一半稳定？为什么稳定快照仍要检查序号？CPU 与 DMA 数据流分别经过哪些动作？

## 学习笔记

记录 `.ioc` 的 channel、direction、increment、width、count、mode；贴出 memory DMA 比对和快照序号。分开写 automatic host/build、semi-automatic device 值、manual 接线/万用表。未接真实硬件时必须写“待实机验证”，不能把 ELF 链接成功写成 ADC/DMA 已在板上成功。

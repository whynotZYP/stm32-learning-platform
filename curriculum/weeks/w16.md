# 第 16 周：I2C 协议与 MPU6050 身份读取

## 学完后能解释

你能解释 I2C 的 SDA/SCL 为什么是开漏：器件只能主动拉低，逻辑高由外部上拉产生，多个器件才不会一个推高、另一个拉低而短路。两线都需要外部 3.3 V 上拉；“GPIO 配 AF-OD”不等于板上自动出现合适电阻。

SDA 在 SCL 高时由高变低是 START，由低变高是 STOP。START 后发送 7-bit address 加一位 R/W，接收方在第 9 个 clock 拉低 ACK；不拉低是 NACK。HAL 接口的设备地址参数使用左移后的 8-bit 位置，所以 MPU6050 AD0=0 的 7-bit 0x68 要写成 `0x68 << 1`，不是把 0xD0 当作 7-bit 地址。

## 学完后能做到

你能断电连接 PB6/SCL、PB7/SDA、3.3 V、GND，并用两只约 4.7 kΩ 电阻把 SDA/SCL 上拉到 3.3 V。CubeMX 配 I2C1、AF open-drain、100 kHz。上电前确认模块没有把两线拉到 5 V。

本周只读取 MPU6050 `WHO_AM_I`：寄存器 0x75，期望 0x68。固件调用 `HAL_I2C_Mem_Read`，address 为 `0x68 << 1`，timeout 20 ms；结果明确分类为 OK、TIMEOUT、BUS_ERROR、WRONG_ID。加速度、陀螺仪、量程和数据换算留到第 18 周。

## 概念模型

一次 register read 可画成：START → address+W → ACK → register 0x75 → ACK → repeated START → address+R → ACK → data 0x68 → master NACK → STOP。写阶段告诉从机“内部寄存器指针”，读阶段取值；repeated START 保持总线事务连续。

NACK 可能来自错误地址、器件未供电、未共地或时序问题；SDA/SCL 一直低可能来自接线短路或器件卡住。错误原因不能都叫“读不到”，程序至少区分超时、其他 HAL 总线错误和收到但 ID 错。

## CubeMX 为什么这样配

目标仍为 STM32F103C8Tx、8 MHz HSE、72 MHz。I2C1 默认在 PB6/PB7，两脚由 HAL MSP 生成 `GPIO_MODE_AF_OD`，速率 100000、duty cycle 2、7-bit addressing、clock stretching enable。外部上拉属于电路，不由 `.ioc` 代替。

本周使用 HAL 的有界 memory read，而不手写无限等待 SR1/SR2 flag 的循环。`App_Init` 只读一次 ID，`App_Run` 为空，避免在错误线路上高频重试占满 CPU 和总线。

## 最小实验

先在纸上画完整 read sequence，并在每个 byte 后标 ACK/NACK。断电完成 PB6/PB7、外部 3.3 V 上拉、模块 3.3 V 和公共地。若模块 AD0 接地，地址为 0x68；若 AD0 拉高则地址改变，本项目期望 AD0=0。

上电观察 `App_GetWhoAmIValue()` 和 `App_GetWhoAmIResult()`。真实连接正确时应在 20 ms 内得到 0x68/OK。没接设备时 TIMEOUT 或 BUS_ERROR 都是诚实结果；host 分类测试通过不能替代真实器件证据。

## 调试与寄存器观察

查看 GPIOB CRL 中 PB6/PB7 的 alternate-function open-drain，I2C1 CR1 的 PE，CCR/TRISE 对应 100 kHz，SR1 的 SB、ADDR、BTF、RXNE、AF 和 TIMEOUT，SR2 的 BUSY。逻辑分析仪可对应 START、地址、ACK、寄存器和数据，但波形仍需人工解释。

若 BUSY 永不清，先断电检查 SDA/SCL 是否被拉低。若地址 NACK，确认传给 HAL 的是左移值 0xD0，而纸面分析仍称 7-bit 地址 0x68；两种表示不能混用。

## 故障注入

第一项把 `0x68 << 1` 错写为把 0xD0 再左移，观察 NACK/error 并恢复。第二项断电后移除外部上拉，再上电观察 TIMEOUT/BUS_ERROR 必须在 20 ms 内返回，随后断电恢复。第三项保留通信但把 expected ID 改错，证明 WRONG_ID 与总线失败不同。

所有改线前必须断电；上拉只能到 3.3 V；模块必须共地。不要把一次读 ID 扩展成六轴数据项目。

## 复述检查

为什么开漏必须有上拉？START/STOP 在什么电平条件发生？第 9 个 clock 是什么？7-bit 0x68 和 HAL 参数 0xD0 各用于哪里？寄存器读为何先写 0x75？最后一个 byte 为何由 master NACK？TIMEOUT、BUS_ERROR、WRONG_ID 分别说明什么？

## 学习笔记

画 START 到 STOP 全序列；记录 PB6/PB7、上拉阻值、100 kHz、地址两种表示、寄存器和期望值。把 host/build、device ID、manual wiring/voltage 分开。没有实机时只报告待验证，不把编译成功写成 MPU6050 已响应。

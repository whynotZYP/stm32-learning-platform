# 第 21 周：RTC、Unix UTC 与备份域

本周把“时间”从普通变量提升为断开主电源后仍应保持的系统状态。安全边界：改接 LSE、VBAT 或电源前必须断电，VBAT 不得反接。本文中的 LSE 起振、VBAT 续时和物理断电结果均为**待实机验证**。

## 学完后能解释

- LSE 是 32.768 kHz 低速外部晶振；预分频 `32767` 后，RTC 计数器每秒加一。
- VBAT 给备份域供电；BKP 寄存器与 RTC 计数器属于备份域，不等于普通 SRAM。
- 本项目把 32 位计数解释为 Unix UTC 秒，不把本地时区写入 RTC；时区只在显示层转换。

## 学完后能做到

- 只在 `BKP_DR1` 缺少 `CLOCK_BACKUP_MARKER` 时设置初始 Unix UTC。
- 普通复位不执行 backup reset、`BDRST` 或强制清除备份域。
- 用 high-low-high 读取 32 位计数：若前后高 16 位不同，重读低位直至高位稳定。

## 概念模型

`LSE -> 32767 prescaler -> RTC counter -> Unix UTC`。主电源复位只重启 CPU；VBAT 存在时，RTC 与 BKP 继续。BKP marker 表示“已完成可信初始化”，不是“时间一定正确”的证明。生产逻辑把 high-low-high 封装为可注入 reader，主机测试会模拟低位从 `0xffff` 翻转到 `0x0002`。

## CubeMX 为什么这样配

选择 `RCC_RTCCLKSOURCE_LSE`，因为它比 LSI 更适合时间保持；`RTC.AsynchPrediv=32767` 对应一秒节拍。`rtc.c` 启用 PWR/BKP 访问和 RTC clock，但应用层不重置 backup domain。若板上没有合适 LSE 或 VBAT，配置能编译也不能说明硬件可用。

## 最小实验

1. 构建 `firmware/lessons/w21-rtc-bkp` 并运行 host behavior。
2. 首次启动观察 marker 缺失路径写入默认 UTC 与 marker。
3. 再次软件复位，确认 marker 路径不改写 counter。
4. 设备实验：记录时间、断开主电源、等待、复电比较秒差；此步骤当前待实机验证。

## 调试与寄存器观察

观察 `RCC_BDCR` 的 LSE ready 与 RTC source、`RTC_CNTH/RTC_CNTL`、`BKP_DR1`。先记录值再决定是否清标志。调试器暂停 CPU 不一定暂停 RTC，因此断点期间时间继续是合理现象。自动证据只有 ELF、静态配置和 host fixture，不包含晶振波形。

## 故障注入

- 主机 fixture 在第一次 high-low-high 中制造跨秒，必须返回第二次稳定值 `0x12350002`。
- 把 marker 改错，确认只触发一次显式初始化；禁止用清整个备份域来“修复”。
- 拔掉 LSE 或 VBAT 的实验必须断电操作，结果记录为待实机验证，不能从仿真推断。

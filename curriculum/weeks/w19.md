# 第 19 周：SPI、W25Q64 与软件 SPI

## 学完后能解释

SPI 用 SCK 提供节拍，MOSI 与 MISO 同时移位，CS 选择从设备。四种模式由 CPOL/CPHA 组合而成：mode 0=`0/0`（低电平空闲、第一沿采样），mode 1=`0/1`（低电平空闲、第二沿采样），mode 2=`1/0`（高电平空闲、第一沿采样），mode 3=`1/1`（高电平空闲、第二沿采样）。W25Q64 本实验使用 mode 0：上升沿采样，数据 MSB first。软件 SPI 将 PB12=CS、PB13=SCK、PB14=MISO、PB15=MOSI 配成普通 GPIO；每一位先设置 MOSI，再把 SCK 拉高并读取 MISO，最后拉低 SCK。一次命令的 command/address/data 期间 CS 必须持续为低，不能在字节之间抬高。

`0x9F` 读取 JEDEC ID，本实验只接受 Winbond W25Q64 的 `EF 40 17`。写操作前必须发 `WREN(0x06)`，随后读状态寄存器确认 WEL 位为 1；擦除或页编程后轮询 WIP 位，以 wrap-safe 毫秒时间判断期限：页编程 10 ms、扇区擦除 500 ms，超限返回 timeout。页编程不能跨 256 B page，扇区擦除按 4 KiB sector 对齐。工程没有 chip erase、mass erase、bootloader 或任意网页地址入口。

## 学完后能做到

能画出 mode 0 的空闲、改变和采样边沿；能用软件 GPIO 完成一整段 CS transaction；能读取并核对 `EF4017`；能在 WREN 后检查 WEL，在写后有界等待 WIP。能拒绝跨页、跨扇区和保留区外地址。能说明为什么固定测试地址 `0x007FF000` 必须集中定义，以及为什么写前保存完整 4 KiB、测试后恢复完整 4 KiB并回读才算流程结束。

## 概念模型

SPI 像四人协作：CS 是点名，SCK 是口令，MOSI 是主机发出的队列，MISO 是从机同时返回的队列。模式错一位就像听错口令时机，数据会整体错位。NOR Flash 的编程只能把 1 变 0，想把 0 变 1 必须先擦除整个 4 KiB 扇区；所以 256 B 的“小改动”也可能伤到同扇区其他内容，必须先完整备份再恢复。

## CubeMX 为什么这样配

本周故意不开 SPI2 外设。PB12/CS 推挽输出且默认高，PB13/SCK 推挽输出且默认低，PB14/MISO 输入无上下拉，PB15/MOSI 推挽输出且默认低，速度为 High。`SoftSpi_Init` 建立 mode 0 空闲状态。应用只在调试者明确调用 `App_RequestFixedSectorTest()` 后执行写测试，复位不会自动反复擦写。固件通过 `HAL_GetTick` 记录全过程 elapsed time，后续与硬件 SPI 比较。

## 最小实验

1. 断电后连接 W25Q64：3.3 V、GND、PB12/CS、PB13/CLK、PB14/DO、PB15/DI，所有模块共地；严禁把 3.3 V Flash 接到 5 V。
2. 下载 `w19-software-spi-w25q64`，先读取 JEDEC ID，只有 `EF4017` 才继续；其他 ID 立即停止写操作。
3. 在调试器中调用 `App_RequestFixedSectorTest()`；程序先保存 `0x007FF000` 起始的完整 4 KiB，再擦除、写入固定 256 B 测试页并回读。
4. 无论测试写成功与否，只要破坏性操作已开始，都执行整扇区恢复；恢复后按 256 B 分块回读比较。任何恢复失败都使最终结果失败。
5. 保存 CS/SCK/MOSI/MISO 波形、状态结果与耗时；未连接真实 Flash 时标记“待实机验证”。

## 调试与寄存器观察

先看 CS 是否从命令开始一直保持低到最后一个数据位结束，再看 SCK 是否低电平空闲、上升沿采样。ID 为全 `FF` 常见原因是 CS 未拉低、MISO 浮空或器件未供电；全 `00` 常见原因是线路短路或模式错误。写不动时按顺序查 WREN transaction、状态 WEL、命令地址、WIP 是否归零。地址日志必须全部位于 `0x007FF000–0x007FFFFF`，页编程长度不超过 256 B 且不跨页。

## 故障注入

把采样改到下降沿，预期 JEDEC ID 不再稳定；在命令与地址间抬高 CS，预期从机把它们解释为不同 transaction；省略 WREN，预期 WEL 检查返回专用错误；让 fake WIP 始终为 1，预期到 10/500 ms deadline 后返回 timeout。尝试 `0x007FF0FF` 写 2 B 或测试扇区外地址，必须在发总线命令前返回 bounds。恢复阶段故意篡改一个字节，最终必须报告 restore failure。

## 复述检查

mode 0 的 CPOL/CPHA 各是多少？数据在哪个边沿采样？为什么 CS 不能每个字节抬高？`9F`、`06`、`05`、`20`、`02` 各做什么？WEL 与 WIP 分别表示什么？页和扇区边界是多少？为什么只写 256 B 仍要保存与恢复完整 4 KiB？为何恢复失败不能被原测试成功掩盖？

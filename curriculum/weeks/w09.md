# 第 9 周：输出比较与 1 kHz PWM

## 学完后能解释

定时器不仅会在计数器溢出时产生事件，还能把 `CNT` 与比较寄存器 `CCR1` 比较，并把结果送到输出通道。PWM 是这种输出比较的周期性用法：在 PWM mode 1、向上计数且高电平有效时，`CNT < CCR1` 输出有效，`CNT >= CCR1` 输出无效。TIM2 时钟为 72 MHz，设置 `PSC=71` 得到 1 MHz 计数时钟，`ARR=999` 形成 1000 个计数，所以 `72 MHz/(71+1)/(999+1)=1 kHz`。占空比近似为 `CCR1/(ARR+1)`，因此 CCR1=350 对应 35%。

输出比较模式、极性 polarity、比较值和引脚路径必须同时正确。`TIM2_CH1` 并不会自动出现在任意 GPIO；本实验使用 PA0 的 alternate function 复用路径。预装载 preload 把新的 CCR1 暂存在影子寄存器，到更新事件才统一生效，避免主循环恰好在周期中间修改比较值而制造窄脉冲。寄存器观察可看 `TIM2_PSC`、`TIM2_ARR`、`TIM2_CCR1`、`TIM2_CCMR1.OC1PE` 和 `TIM2_CCER.CC1P/CC1E`。SPL 的输出比较配置概念在 HAL 中对应 `TIM_OC_InitTypeDef`、`HAL_TIM_PWM_ConfigChannel` 和 `HAL_TIM_PWM_Start`，不是逐个函数的一对一替换。

## 学完后能做到

能从定时器输入时钟推导 1 kHz 的 PSC/ARR，选择 TIM2_CH1 到 PA0 的复用路径，启用 PWM1、高电平极性和比较预装载。能每 100 ms 把 CCR1 以 100 计数为一步在 0 到 1000 之间往返改变，形成可观察的 LED 呼吸，而不使用 `HAL_Delay` 阻塞主循环。能明确区分“代码构建成功”“寄存器设定正确”和“真实引脚频率正确”三类证据。

## 概念模型

把定时器想成循环走动的刻度尺。ARR 决定一圈有多少刻度，CCR1 决定这一圈前多少刻度保持有效电平，极性决定“有效”是高还是低，通道复用决定这个内部信号能否走到封装引脚。频率由一圈走多快决定，占空比由有效区占一圈的比例决定。preload 像在下一圈开始前准备好新刻度，更新事件到来时再一次换上，因此波形边界整齐。

## CubeMX 为什么这样配

芯片选 STM32F103C8Tx，HSE 8 MHz 经 PLL 得到 72 MHz，APB1 为 36 MHz；因为 APB1 预分频不为 1，TIM2 的实际时钟仍为 72 MHz。TIM2 选择 Internal Clock、Channel 1 PWM Generation，Prescaler 71、Counter Period 999、Pulse 初值 0、PWM mode 1、高电平有效；PA0 自动成为 TIM2_CH1 复用推挽输出。App 启动前显式启用 OC1 preload，再启动 PWM。改接 PA0、LED 或测量探头前必须断电；逻辑电平只用 3.3 V，探头与开发板共地，LED 串联 1 kΩ 电阻。GPIO 不能直接驱动舵机、电机等大负载。

## 最小实验

1. 断电后把 PA0 经 1 kΩ 电阻串联 LED 到 GND，确认 LED 极性和公共地后上电。
2. 构建并下载 `w09-pwm`，在 `App_Init` 后确认 `HAL_TIM_PWM_Start` 返回成功。
3. 观察 `PSC=71`、`ARR=999`、`CCR1` 每 100 ms 以 100 为步长变化；CCR1 到 1000 后反向，回到 0 后再上升。
4. 人工观察 LED 由暗到亮再到暗。亮度只是人工证据；真实 PWM 数值留到第 11 周用 PA0→PA6 回环测量。
5. 若有示波器，可额外记录约 1 kHz 波形，但没有仪器时必须写“待实机验证”，不能拿 ELF 代替物理测量。

## 调试与寄存器观察

暂停在主循环，核对 `TIM2->PSC=71`、`TIM2->ARR=999`、`TIM2->CCR1` 的范围，并观察 `CCMR1.OC1M` 为 PWM1、`CCMR1.OC1PE` 已开启、`CCER.CC1E` 允许输出。若 CNT 正常增加但 PA0 没波形，沿“定时器通道→AFIO/GPIO 复用→PA0→电阻→LED→GND”逐段排查。若频率差一倍，优先检查 APB1 定时器倍频；若占空比边界有毛刺，检查比较预装载和更新事件，而不是盲目增加 delay。

## 故障注入

先断电，再在 CubeMX 中把 PA0 改成普通 GPIO，或在受控代码中暂时关闭 `OC1PE`。重新构建后，普通 GPIO 会切断 timer channel path；无 preload 时在周期中途改 CCR1 可能出现不规则窄脉冲。记录症状、寄存器差异和因果解释，再恢复 TIM2_CH1 alternate function 与 preload。故障实验仍只使用 3.3 V 和共地，不允许带电插拔探头或 LED。

## 复述检查

请不看代码回答：为什么 72 MHz、PSC=71、ARR=999 得到 1 kHz？为什么公式里两处都要加 1？CCR1=250 时占空比是多少？PWM1、高电平极性、CC1E、PA0 复用和 preload 分别解决哪一段问题？最后说明为什么“LED 看起来在呼吸”不能精确证明 1 kHz，而构建成功更不能证明真实引脚有波形。

记录时钟树假设、PSC/ARR/CCR1 推导、PA0 通道路径、寄存器快照、一次错误复用或 preload 故障的修复。证据分为 automatic 构建/静态检查、semi-automatic 设备寄存器或后续回环数值、manual LED 亮度观察。未连接开发板或仪器时明确写“待实机验证”，不要把模拟器或编译结果写成物理通过。

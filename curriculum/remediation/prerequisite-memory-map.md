# 内存映射与 volatile 先修补救

## 目标

预计时间：30 分钟。能从外设基地址和偏移定位寄存器，并解释为什么硬件寄存器访问需要 `volatile`。

## 解释

内存映射把外设寄存器放进 CPU 地址空间；基地址选择外设，偏移选择寄存器。硬件可在程序之外改变值，所以编译器不能把读取结果长期缓存。

## 微型练习

给定基地址 `0x40010800` 和偏移 `0x0C`，计算目标地址；画出指针、地址、寄存器值和物理引脚之间的关系。

## 检查问题

如果轮询代码没有使用 `volatile`，编译器可能怎样处理重复读取，最终会出现什么可观察现象？

## 通过证据

提交地址计算、关系图和一段只读寄存器的最小 C 表达式，并指出其中不能被优化掉的读取。

# STM32 课程共同评分量表

总分 = 概念理解 25% + 配置与工具 25% + 实操作品 35% + 复盘表达 15%。每类先按下表得到 0、25、50、75 或 100 的等级分，再乘以该类权重；没有可检查证据时不得只凭口头声明给分。

## 概念理解（25%）

| 评分 | 可观察锚点 |
| --- | --- |
| 0 | 没有作答，或描述与本题外设、信号和寄存器关系无关。 |
| 25 | 能指出一个相关名词，但无法用自己的话说明输入、状态变化和输出。 |
| 50 | 能画出基本数据流并解释主要步骤，但遗漏一个关键条件或标志位。 |
| 75 | 能完整解释数据流、关键寄存器或标志，并预测一个常见故障的现象。 |
| 100 | 能用波形、寄存器快照或代码证据验证解释，并准确比较正常与故障路径。 |

## 配置与工具（25%）

| 评分 | 可观察锚点 |
| --- | --- |
| 0 | 未提交可打开的 CubeMX 配置或工程不能完成配置阶段。 |
| 25 | 工程可打开，但关键引脚、时钟、中断或 DMA 配置与任务不符。 |
| 50 | 主要配置正确且能生成代码，但无法指出配置对应的初始化代码或寄存器。 |
| 75 | 配置、生成代码和调试观察一致，并能独立定位一个配置错误。 |
| 100 | 提交可复现的配置步骤、生成差异和调试记录，能解释每个关键选项的取舍。 |

## 实操作品（35%）

| 评分 | 可观察锚点 |
| --- | --- |
| 0 | 没有可构建产物，或接线违反课程明确的安全要求。 |
| 25 | 工程能构建但核心行为未出现，且没有足够日志、波形或寄存器证据定位原因。 |
| 50 | 核心行为在一个正常场景出现，但边界条件、故障恢复或证据记录不完整。 |
| 75 | 正常、边界和指定故障场景均可复现，提交时间戳、串口、波形或寄存器证据。 |
| 100 | 作品可由他人按步骤复现，并用自动、设备和人工证据分别说明已证明与未证明范围。 |

## 复盘表达（15%）

| 评分 | 可观察锚点 |
| --- | --- |
| 0 | 未提交复盘，或内容无法对应本周目标与证据。 |
| 25 | 只写结果好坏，没有记录假设、失败现象或下一步。 |
| 50 | 能按时间顺序记录一次问题和处理，但证据与结论之间仍有跳步。 |
| 75 | 能用证据解释原因、修复和验证，并写出一个可执行的下一步练习。 |
| 100 | 能区分事实、推断和未验证项，给出可复用的排障规则并链接全部关键证据。 |
